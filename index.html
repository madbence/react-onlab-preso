explaining redux

wat is redux?

  • inspired by Elm
  • flux architecture (fancy name for MVC, "invented" by facebook)
  • immutable data
  • (state, action) -> state
  • deterministic (time traveling, event replay, undo/redo)

redux, the state container

import { createStore } from 'redux';

const store = createStore((state, action) => {
  switch(action.type) {
    case 'TICK':
    return { n: state.n + 1 };
  }
  return state;
}, { n: 0 });

export default store;

connecting redux with react

import { connect } from 'react-redux';

class Ticker extend Component {
  render() {
    return <div>
      {this.props.n}
      <button onClick={this.onClick.bind(this)}>tick</button>
    </div>;
  }
  onClick() {
    this.props.dispatch({
      type: 'TICK'
    });
  }
}

export default connect(state => state)(Ticker);

putting it together

import store from './store';
import Ticker from './ticker';
import { Provider } from 'react-redux';
import { render } from 'react';

render(
  <Provider store={store}>
    {() => <Ticker />}
  </Provider>,
  document.getElementById('some-element-id')
)
